Server Specific Settings

Setting

Meaning

Default/Example

service.rmi.port

The default port used by gratia for internal rmi use

17000

The registered rmi name for rmi access

//Fermi.fnal.gov:17000

service.rmi.lookup

The “external” rmi name that outside services can access

rmi://Fermi.fnal.gov:17000

Security Settings

Setting

Meaning

Default/Example

service.security.level

Flag indicating what form of security should gratia assume.

0 = No security

1 = Use tomcat ssl

2 = Apache handling security

service.use.selfgenerated.certs

Flag indicating whether or not gratia should generate it’s own certs or use vdt/doe certs

1 = Use self generated certs

service.vdt.cert.file

Absolute path to vdt/doe public cert file

/etc/grid-security/httpcert.pem

Service.vdt.key.file

Absolute path to vdt/doe private key file

/etc/grid-security/httpkey.pem

service.autoregister.pem

Flag indicating whether or not external services (when running ssl) will automatically be trusted. Values are 0 (no) or 1 (yes) – refer to security howto

1

service.open.commection

The open, unsecured url clients can connect to

http://fermilab:8880

service.secure.connection

The ssl enabled url that clients can connect to

https://fermilab:8843
