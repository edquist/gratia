Note: The search/where clause component should be set to either "Everything" (meaning just what it says) or to a component of a sql where clause. As an example, if you wanted to set up a role that just returned information for a particular VO, the where clause would be (ignoring the double quotes) "VOName = 'some value'". A second example: "VOName = 'some value' and ProbeName = 'some value'". The underlying report software will invoke a stored procedure which will create a dynamic sql query to return just those values. Also note that whatever subtitle you enter is displayed at the top of each report.
